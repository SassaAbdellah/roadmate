Overview of Testing :

  1. Unit test: Programmers test their own classes during the coding.
  2. Intergation: After one prototype cycle finish, one person is doing integration test:
    • interface
    • contents
    • security
    • navigtion
    • validation
  3. System test: Test roadmate in Windows , Mac, linux
  4. Usability test: choose different group of users, observe users performance, record users feedback.
  5. Stress test: use tools make large amount requests and record performance data.

This testing process is a whole big cycle including some smal internall cycle. During each prototype design and coding, there is a frequence loop between Unit test and Integration . From system test, usability test, and stress test, we will come back beginning point, change our code and the new cycle start again and again.
